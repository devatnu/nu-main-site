## 6. Technical Implementation Notes

### 6.1 Architecture

Myra is a frontend chat component that calls the Anthropic API (Claude Sonnet) with a pre-loaded system prompt containing all the knowledge above. No backend needed — the API call happens client-side from the React app.

### 6.2 System Prompt Structure

The system prompt should include:
1. Myra's personality and tone guidelines (Section 3)
2. The complete knowledge base (Section 4) as structured context
3. Conversation pattern examples (Section 5) as few-shot guidance
4. A fallback instruction for unknown queries

### 6.3 UI Placement

- There are existing AI buttons in the website, on click Myra triggers.
- Opens into a chat panel/drawer
- First message from Myra on open: "Hey! I'm Myra, Nishant's AI assistant. Ask me anything about his work, experience, or how to get in touch."
- Typing indicator while waiting for API response
- Chat history persists during session (React state), resets on page reload

### 6.4 Suggested Tech Stack

- React component with useState for chat history
- Anthropic API call (Claude Sonnet) — see anthropic_api_in_artifacts documentation
- No external dependencies needed beyond the API call
- Model: claude-sonnet-4-20250514
- Max tokens: 500 (keep responses concise)
- System prompt: paste the full knowledge base + personality guidelines

### 6.5 Privacy and Rate Limiting

- No user data is stored or sent anywhere beyond the API call
- Consider adding a simple rate limit (e.g., max 20 messages per session) to manage API costs
- No authentication required — Myra is public-facing
